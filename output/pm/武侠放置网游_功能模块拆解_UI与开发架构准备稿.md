## 0. 文档目标

这份文档不再重复讲一遍策划案，而是把新版机制资料转换成项目经理可直接使用的研发拆解稿，解决四件事：

1. 把首发版本的范围、优先级和冻结边界说清楚。
2. 把前台信息架构、核心页面、关键状态页拆给 UI。
3. 把服务端逻辑域、核心数据域、异步任务和配置簇拆给研发。
4. 把排期、协作、验收和风险点收拢成统一项目视图。

文档默认基于以下共识：

- 首发采用“1 份总纲 + 11 份模块文档”的设计资产结构。
- 首发版本范围是高安、低安、危险区三层地图，开放 `朔砂外原` 与 `伏龙渊域外围/前哨`。
- 主线开放前三幕，第四幕与 `沉灯海隙` 保留接口，不进入首发研发主线。
- 项目目标是先做“成长 x 经济 x 风险 PvP x 组织政治”的闭环，而不是一次性做完整长线资料片。

## 1. 新版机制输入基线

### 1.1 机制文档结构已经变化

新版机制不是单文档直推研发，而是采用：

- 1 份总纲：负责北极星、首发边界、共享词典、全局公式、命名规范、配置接口约定。
- 11 份模块分册：负责具体玩法、条目库、公式、示例文案、配置表建议、依赖关系。

项目管理上应按这个结构分配任务：

- 总纲作为“统一口径中心”，由 PM、主策、技术负责人共同维护。
- 模块分册作为“研发工作底稿”，按模块负责人分配与迭代。
- 世界观与原始审阅稿作为“叙事与方向约束”，不直接作为研发规则源。

### 1.2 首发已冻结的关键事实

| 类别 | 已冻结口径 |
| --- | --- |
| 地图规模 | 首发 196 个节点，分布在 8 个区域群，分为核心重节点、功能节点、路线/事件轻节点 |
| 地图圈层 | 高安、低安、危险区、事件态 |
| 市场结构 | 4 个市场圈：高安核心圈、边渡水网圈、山关群谷圈、外原前哨圈 |
| 放置底盘 | 4 槽基础队列，24 小时离线结算上限 |
| 行动模板 | 首发 10 类具体行动模板，归属 8 个母类 |
| 挂机 PvE | T1-T6 匪巢，清剿后必须手动“前往下一处” |
| PvP 分层 | 擂台、低安开红、危险区自由 PvP、据点战/帮战 |
| 核心高风险区 | `朔砂外原`、`伏龙渊域外围/前哨` |
| 报告系统 | 江湖日报为每日首次登录主入口 |
| 商业化边界 | 不卖数值，只卖外观、舒适度、功能位便利 |

### 1.3 PM 的阅读顺序建议

为避免多人协作时口径跑偏，建议固定阅读顺序：

1. 先读总纲：明确首发边界、共享词典、统一公式。
2. 再读 02、03、06、08、09 模块：优先掌握地图、队列、市场、PvP、帮派这五个中轴。
3. 再读 01、04、05、07、10、11 模块：补齐成长、物品、制造、PvE、叙事、日报与商业化。
4. 最后回看世界观稿：确认 UI 叙事口吻和区域氛围不偏题。

## 2. 首发产品抽象

### 2.1 一句话产品定义

一款以“节点江湖”为地图骨架、以“行动队列 + 离线收益”为成长底座、以“区域市场 + 风险 PvP + 帮派政治”为长期驱动的武侠放置网游。

### 2.2 玩家体验主张

- 白天 5-15 分钟：收菜、调队列、看日报、查市场、处理挂单。
- 晚上 30-90 分钟：匪巢、远征、副本、护镖、劫镖、擂台、帮战择一或组合。
- 长线目标：通过制造、跑商、成长、情报、据点与恩怨，形成“我在一个活着的江湖里经营与冒险”的感受。

### 2.3 首发体验闭环

```text
行动队列/离线收益
    -> 采集/制造/炼药/打坐
    -> 区域市场挂单与跨圈运输
    -> 护镖/劫镖/危险区刷匪/帮派情报
    -> 货物掉落、装备损毁、税费回收
    -> 再回到生产、补给、构筑优化
```

### 2.4 首发成功标准

产品和研发需共同盯住以下目标：

- D1 留存不低于 40%。
- D7 留存不低于 20%。
- D30 留存不低于 10%。
- 周银两净注入保持在健康区间，不出现明显通胀失控。
- 危险区平均净收益高于高安，但不会高到逼迫所有人都下危险区。
- 帮派情报、护镖/劫镖、市场价差和匪巢密报必须形成真实可玩的联动。

### 2.5 首发不做什么

- 不做第四幕完整玩法。
- 不做 `沉灯海隙` 资料片级内容。
- 不做完整赛季战争。
- 不做重微服务分布式架构。
- 不做复杂职业锁死和多套独立挂机系统。

```text

  -> 制造采集与经济
  -> 市场交易与运输

  -> 帮派社交与政治经济

  -> 日报榜单活动与商业化
```

### 3.3 三层产品结构

#### A. 经营成长层

- 行动队列与离线
- 角色与成长
- 物品与资产
- 制造采集与经济
- 首页与日报

#### B. 风险流通层

- 地图与路线
- 市场交易与运输
- PvE 玩法
- PvP 与风险结算

#### C. 组织政治层

- 帮派社交与政治经济
- 叙事主线与任务
- 日报榜单活动与商业化

PM 管理上要确保三层都“有首发闭环”：

- 只有经营成长层，没有风险流通层，市场就会变成库存堆积。
- 只有风险流通层，没有组织政治层，深度玩家会缺长期目标。
- 只有组织政治层，没有经营成长层，新玩家与休闲玩家会直接流失。

### 4.1 一级导航建议

1. 首页
2. 地图
3. 行动
4. 角色
5. 仓库

## 5. 功能模块详细拆解

### 5.1 账户与接入模块

- 业务目标：支撑 Steam/PC 首发登录、角色创建、公告、邮件、设置与基础状态管理。
- 子模块：登录鉴权、选服/选角、角色创建、命名校验、协议与设置、邮件公告。
- 核心页面：启动页、登录页、角色创建页、选角页、邮件公告页、设置页。
- 关键实体：`Account`、`CharacterProfile`、`Session`、`MailMessage`、`PlayerSettings`。
- 服务端职责：账户鉴权、角色基础资料、命名服务、邮件公告、版本校验。
- PM 注意点：这是所有玩法底座，必须优先完成“可追溯角色状态”与“系统公告入口”。

### 5.2 首页与日报模块

- 业务目标：作为玩家每日进入游戏后的第一主入口，承接世界变化、个人总结和当日目标引导。
- 核心页面：日报首页、个人摘要、帮派简报、增刊快讯。
- 关键实体：`JianghuDailyEntry`、`BattleNews`、`DailyTaskDigest`。
- 依赖：行动队列、市场、帮派、叙事、榜单。
- PM 注意点：日报不是装饰层，而是把“昨晚发生了什么、你现在该做什么”讲清楚的总入口。

### 5.3 地图与路线模块

- 业务目标：把 38 节点、4 市场圈、风险路线、税率和驻防，做成可读可判定的世界主界面。
- 核心页面：世界地图、节点详情、路线预览、热区/情报图层。
- 关键实体：`NodeRegion`、`RouteEdge`、`SecurityBand`、`TerritoryNode`。
- 关键配置：节点开放规则、路线耗时、热度、痕迹、税率、事件封锁。
- PM 注意点：地图是市场、运输、匪巢、帮派、主线的统一入口，不可当成附属页面开发。

### 5.4 行动队列与离线模块

- 业务目标：形成留存基本盘。
- 首发关键参数：4 槽队列、24 小时离线结算、10 类行动模板、匪巢手动下一处。
- 核心页面：队列总览、行动配置、离线结算、中断恢复、匪巢结算。
- 关键实体：`ActionQueueSlot`、`OfflineSettlementRecord`、`InterruptReason`。
- PM 注意点：必须把“为什么停”“如何恢复”“现在拿到了什么”显示清楚，否则用户会把策略系统感知成卡死。

### 5.5 角色与成长模块

- 业务目标：承接技能、领悟、心法、装备、行具与构筑差异。
- 核心页面：属性页、技能树、领悟页、装备页、心法页、行具页、构筑预设。
- 关键实体：`CharacterBuild`、`SkillTrack`、`EquipmentItem`、`ManualLoadout`、`VehicleSlot`。
- PM 注意点：必须让玩家同屏理解战斗收益、生活收益、负重、跑商能力与风险承受能力。

### 5.6 物品与资产模块

- 业务目标：统一承接背包、仓库、战利品栏、货仓、耐久、分解、绑定与损毁后的资产迁移。
- 核心页面：背包、仓库、战利品栏、货仓、修理弹窗、分解弹窗。
- 关键实体：`ItemInstance`、`InventorySlot`、`WarehouseStorage`、`CargoStorage`、`DurabilityState`。
- PM 注意点：容器概念必须清晰，否则用户会持续搞不清“货物、战利品、材料、装备”去哪了。

### 5.7 制造采集与经济模块

- 业务目标：形成玩家经济底座。
- 核心内容：采集、蓝图、主材槽、制造、分解、材料来源矩阵、银两排水口。
- 关键实体：`BlueprintRecipe`、`CraftJob`、`MaterialSource`、`EconomicSinkRecord`。
- PM 注意点：高价值配方数量应控制，先做“可感知差异”的主材槽，不做大量同质蓝图。

### 5.8 市场交易与运输模块

- 业务目标：让区域价差、护镖、劫镖、保险和税费成为真实玩法，而不是数值说明。
- 核心内容：4 市场圈、4 类运输订单、重点贸易商品、挂单、合同、保险、税费分账。
- 关键实体：`MarketListing`、`MarketTradeRecord`、`TransportOrder`、`TaxSnapshot`。
- PM 注意点：市场要同时满足普通买卖和深度倒货两类诉求，首发不做复杂量化 UI，但必须把利润预估做清楚。

### 5.9 PvE 模块

- 业务目标：提供稳定产出、挂机惊喜和主动内容梯度。
- 核心内容：T1-T6 匪巢、命名精英、密报、远征、副本、世界 Boss。
- 关键实体：`BanditNestInstance`、`LootBundle`、`ExpeditionClue`、`DungeonInstance`、`BossEvent`。
- PM 注意点：匪巢和密报是首发最重要的“放置 -> 主动”桥梁。

### 5.10 PvP 与风险结算模块

- 业务目标：让 PvP 成为经济消耗出口和路线冲突驱动力。
- 核心内容：低安开红、危险区自由 PvP、护镖/劫镖、掉货、损毁、保险、护命、通缉、保护期。
- 关键实体：`PvPKillResult`、`InsurancePolicy`、`WantedRecord`、`ProtectionState`。
- PM 注意点：要把“货物掉落”和“装备损毁”清晰分层，避免普通玩家误解为全身爆装。

### 5.11 帮派社交与政治经济模块

- 业务目标：让帮派成为安全感来源、情报网络载体和危险区长期目标承载体。
- 核心内容：岗位、情报、据点、税率、维护费、恩怨、师徒、商誉、帮派焦点活动。
- 关键实体：`Guild`、`GuildRole`、`GuildIntelReport`、`TerritoryNode`、`GuildWarRecord`、`GrudgeRecord`。
- PM 注意点：若帮派只剩聊天和加成，就会失去新版机制里最核心的组织价值。

### 5.12 叙事主线与任务模块

- 业务目标：用前三幕 9 章把系统解锁、节点波次开放和区域叙事顺滑地串起来。
- 核心内容：章节解锁、任务链、人物关系、势力关系、节点开放、余波任务、日报播报。
- 关键实体：`MainChapter`、`Quest`、`NPCState`、`ChapterUnlockRule`。
- PM 注意点：主线页必须承担“系统为什么现在开放”的说明工作。

### 5.13 日报榜单活动与商业化模块

- 业务目标：提供身份反馈、运营节奏和不卖数值的收入边界。
- 核心内容：日报结构、5 类首发榜单、每周焦点活动、每月世界事件、商城条目、公平声明。
- 关键实体：`RankingRecord`、`EventCampaign`、`ShopSku`、`ConvenienceEntitlement`。
- PM 注意点：公平声明和商城表达要前置进 UI 与运营文案，不要上线前临时补。

### 5.14 配置中心、风控与后台模块

- 业务目标：支撑调参、排障、反刷、补偿和 live 运营。
- 核心内容：配置热更、日志审计、风控规则、GM 后台、周平衡看板。
- 关键实体：`ConfigVersion`、`DropPoolConfig`、`WorldEventConfig`、`RiskAuditLog`、`BalanceDashboard`。
- PM 注意点：这部分不面向玩家，但决定首发能否稳定运营和快速修问题。

首发建议采用“模块化单体 + 统一异步任务中心 + 配置中心”，不建议直接做重型微服务。

| 逻辑域 | 主要职责 | 对外依赖 |
| --- | --- | --- |
| AccessDomain | 登录、鉴权、角色入口、邮件公告 | Account、Player |

| WorldDomain | 节点、路线、开放状态、落点判定、战争态 | Guild、Narrative、Economy |
| GrowthDomain | 属性、技能、领悟、心法、行具、构筑 | Asset、PvE、PvP |
| QueueDomain | 行动计划、离线结算、异常恢复 | Growth、World、Asset |
| AssetDomain | 物品、仓储、耐久、分解、货仓 | Growth、Economy、PvP |
| EconomyDomain | 市场、挂单、订单、运输、税费、估值 | Asset、World、Guild |
| PvEDomain | 匪巢、副本、掉落、密报、远征 | Growth、Asset、Narrative |
| PvPDomain | 开红、掉货、损毁、保险、通缉、保护期 | Asset、World、Guild |
| GuildDomain | 帮派、岗位、情报、据点、税率、战报 | World、Economy、PvP |
| NarrativeDomain | 主线、任务、日报、章节解锁、世界事件 | World、Player、PvE |
| OpsDomain | 商城、活动、榜单、埋点、风控、GM 后台 | 全域 |

### 6.3 异步任务与定时任务建议

必须有统一任务调度器，负责：

- 行动队列结算
- 离线收益结算
- 匪巢完成后的掉落与痕迹生成
- 密报过期处理
- 市场挂单过期
- 保险参考价更新
- 每日江湖日报汇总
- 榜单周期结算
- 据点维护费扣除
- 世界事件开关切换
- 风控扫描与异常审计

### 6.4 统一事件总线建议

建议从首发开始统一事件命名：

- `ChapterUnlocked`
- `QueueStarted`
- `QueueInterrupted`
- `QueueCompleted`
- `BanditNestFinished`
- `LootClaimed`

- `MarketTradeFinished`
- `TransportOrderAccepted`

- `TerritoryOwnerChanged`

PM 要把这些事件视为跨模块协作清单，而不是纯技术实现细节。

## 8. 研发排期映射建议

### 8.1 P0：范围冻结与底座搭建

目标：把“做什么、不做什么、怎么拆”一次说清。

交付项：

- 首发范围冻结
- 统一词典与实体模型冻结
- 逻辑域边界冻结
- IA、页面清单、组件清单冻结
- 配置簇模板建立
- 关键风险评估：市场一致性、离线结算、掉货/赔付可追溯

### 8.2 P1：单人闭环原型

目标：实现“单人收菜 -> 制造/匪巢 -> 整理 -> 再排队”的闭环。

交付项：

- 账户与角色基础链路
- 地图基础展示与节点进入
- 队列与离线结算
- 角色成长与装备构筑
- 背包/仓库/战利品栏
- 基础采集、制造、炼药
- 基础匪巢 PvE
- 首页/日报原型

### 8.3 P2：联网经济闭环

目标：实现“生产 -> 市场 -> 运输 -> 价差 -> 日报反馈”的联网闭环。

交付项：

- 4 市场圈与区域挂单
- 成交与税费
- 合同与运输订单
- 基础保险
- 主线前半段系统解锁
- 基础帮派与情报频道

### 8.4 P3：风险与组织闭环

目标：实现“经济产出 -> 风险对抗 -> 装备消耗 -> 组织协作 -> 节点政治”的深度闭环。

交付项：

- 低安/危险区 PvP
- 掉货、损毁、保险赔付、通缉、保护期
- 护镖/劫镖
- 命名精英、密报、远征
- 帮派岗位、据点、税率
- 榜单与战报

### 8.5 P4：内容打磨与运营能力

目标：把前三幕、活动、运营能力和反骚扰策略补齐。

交付项：

- 前三幕主线完整串联
- 副本/世界 Boss 完整化
- 商城与活动
- 周平衡看板
- GM/运营后台
- 新手保护与反骚扰打磨

### 8.6 P5：测试与上架准备

目标：收敛风险、补齐内容边界、准备上架与首轮运营。

交付项：

- 全链路回归
- 经济压测与恢复演练
- 配置热更演练
- 风控与申诉流程演练
- 上架素材、商店说明、公平声明

## 9. UI、研发、测试协同交付建议

### 9.1 UI 侧交付顺序

UI 建议按以下顺序推进：

1. 视觉底板：世界观关键词、色调、字号、面板规范。
2. 设计系统：节点卡、物品卡、行动槽、榜单条、风险标签。
3. 首屏链路：首页/日报、地图、行动、角色、仓库、市场。
4. 风险链路：开红、掉货、损毁、护镖、情报。
5. 组织链路：帮派、据点、税率、战报。
6. 运营链路：榜单、活动、商城。

### 9.2 客户端侧交付重点

- 地图状态层与图层切换
- 队列状态机与异常页
- 匪巢结算与稀有弹窗
- 构筑页复杂信息压缩展示
- 市场与趋势图基础能力
- 战报、日报、榜单等长列表性能

### 9.3 服务端侧交付重点

- 队列与离线结算正确性
- 市场与订单一致性
- 掉货/损毁/保险赔付可追溯
- 情报、战报、日报的异步稳定性
- 风险审计与日志留存

### 9.4 QA 侧交付重点

重点先验证以下链路：

1. 主线解锁是否正确驱动系统开放。
2. 行动队列在离线、热更、中断恢复时是否稳定。
3. 市场挂单、成交、运输、掉货与保险是否一致。
4. 低安/危险区死亡结算是否符合规则。
5. 帮派情报、据点税率、维护费、战报是否闭环。
6. 日报和榜单是否正确汇总世界与个人数据。

## 10. 验收清单与项目风险

### 10.1 八类玩家场景验收

| 场景 | 验收目标 |
| --- | --- |
| 纯休闲收菜 | 每日 15 分钟也能推进主线和成长 |
| 独狼生产 | 不入帮也可在高安和低安短线稳定盈利 |
| 帮派后勤 | 能通过制造、修理、运输持续获得组织价值 |
| 跑商护镖 | 价差、风险、报酬形成闭环 |
| 猎人截杀 | 能通过路线与情报抓到高价值目标 |
| 重甲速刷 | 收益高、损失也高，风险回报明确 |
| 远征小队 | 密报有价值，远征产出匹配难度 |
| 市场操盘 | 能通过信息和节奏获利，而不是只靠刷材料 |

### 10.2 非功能验收

- 所有关键结算有服务端日志可追溯。
- 关键配置支持热更与版本号回查。
- 保险估值能过滤明显对刷与异常成交。
- 重复击杀收益衰减必须生效。
- 关键日报与榜单生成不能阻塞主链路登录。

### 10.3 当前主要研发风险

| 风险 | 影响 | 应对建议 |
| --- | --- | --- |
| 地图、市场、PvP 三者耦合高 | 容易一改动牵全局 | 先冻结共享词典和事件名 |
| 离线结算复杂 | 易出错、易被刷 | 统一快照机制与审计日志 |
| 市场估值被刷单污染 | 保险和回收口会失真 | 做异常样本过滤和审计 |
| UI 信息密度过高 | 新手理解门槛高 | 首页/地图/行动三页优先降噪 |
| 帮派系统做轻了 | 深度玩家无组织目标 | 情报、据点、税率至少做成闭环 |
| 主线与系统解锁脱节 | 玩家会迷路 | 章节页必须承担“为什么开放”的说明 |

### 10.4 仍需项目层冻结的事项

这些不是机制未定义，而是项目执行前仍要明确：

- 客户端技术栈与 UI 框架选型
- 服务端具体语言与数据库方案
- Steam 首发是否预留游客态/试玩态
- 埋点平台与日志平台选型
- 配置表管理方式：表格驱动还是后台驱动
- 文案、插画、地图节点图标的产能节奏

## 11. 建议的下一步产出

基于当前新版机制，PM 最适合继续推进的交付物有 4 份：

1. `功能清单与优先级排期表`：把本文转成 Jira/飞书/Linear 可导入任务。
2. `页面清单与线框需求表`：按页面域拆 UI 任务。
3. `配置表目录与字段模板`：让系统策划和服务端对齐建表。
4. `联调验收用例清单`：把 8 类玩家场景拆成测试用例。

如果项目要正式进入研发，这四份产物的优先级应高于继续追加新玩法说明。

## 12. 结论

新版机制最大的变化，不是“多了几个系统”，而是已经从一份总策划案，升级成了“总纲 + 模块化机制文档”的研发资产体系。

因此，项目管理方式也要同步升级：

- 不再按“整份策划案”推进，而要按模块负责人推进。
- 不再按“页面感觉”出 UI，而要按业务域和状态机出 UI。
- 不再把风险放到上线前处理，而要在队列、市场、掉货、保险、情报这些核心闭环里前置考虑。

只要我们按这份拆解稿推进，首发研发的边界、顺序和依赖关系就是清晰的，团队可以直接进入排期、分工和建表阶段。
