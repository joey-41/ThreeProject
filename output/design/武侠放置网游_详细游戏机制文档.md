# 武侠放置网游详细游戏机制文档

> 文档版本：V1.0  
> 文档角色：首发总纲、统一口径、模块索引、配置接口说明  
> 适用对象：项目经理、系统策划、数值策划、客户端、服务端、UI、测试、运营  
> 适用范围：首发配置级，不覆盖第四幕完整玩法、沉灯海隙完整资料片、完整赛季战争

## 0. 文档定位

本文件不再承担全部机制细节，而是作为首发版本的总纲与统一规则中心，负责：

- 统一产品定位、世界边界、全局公式和共享词典。
- 定义跨模块都要遵守的数值口径、命名方式、文案风格和配置字段。
- 提供模块化详细设计文档的目录索引，便于后续拆给不同策划与研发角色。

首发交付采用“1 份总纲 + 11 份模块子文档”的结构。主文档负责定原则、定边界、定接口；模块文档负责把规则细化到可建表、可实现、可验收的粒度。

## 1. 产品北极星与首发边界

### 1.1 一句话定位

一款以“九河十三路”节点江湖为舞台的武侠放置网游：玩家白天经营生产、挂机清剿、调度交易，晚上参与副本、护镖、劫镖、帮战与区域政治，在玩家驱动的市场里创造自己的江湖传闻。

### 1.2 首发承诺

- 上班挂机，下班主动。
- 市场由玩家驱动，系统只负责底层资源注入与银两回收。
- PvP 必须存在，但安全感略微向生产玩家倾斜。
- 休闲玩家可以靠 15-30 分钟日常推进，深度玩家可以靠路线、情报、构筑与组织获得更高收益。
- 世界观不是包装层，而是规则解释器，地图节点、势力职责、剧情章节都要能解释玩法来源。

### 1.3 首发版本边界

- 地图范围：高安、低安、危险区三层结构开放；危险区首发开放 `朔砂外原` 与 `伏龙渊域外围/前哨`。
- 剧情范围：前三幕完整开放，第四幕只保留入口、伏笔、字段和公告文案接口。
- 节点规模：首发共 38 个节点，分布于 8 个区域群。
- 玩家目标：首发完成“成长 x 经济 x 风险 PvP x 组织政治”的闭环，不追求把全部后期资料片内容一次性塞入。

## 2. 共享词典

### 2.1 核心术语

| 术语 | 定义 |
| --- | --- |
| 节点 | 世界地图上的最小可到达区域单位，玩家所有移动、交易、战斗和驻留都围绕节点发生 |
| 路线 | 节点与节点之间的连接关系，决定移动耗时、风险、税率与追逐博弈 |
| 安全等级 | 高安、低安、危险区、事件态四类规则集合，影响 PvP 合法性、掉落、损毁和收益 |

| 行动队列 | 玩家预设的挂机与离线行动列表，统一承接采集、制造、炼药、打坐、清剿、运输、修理 |
| 匪巢 | 对应刷点核心内容的挂机 PvE 单元，清剿后需要手动前往下一处 |
| 密报 | 由匪巢或事件产出的限时线索，触发远征或高阶副本入口，可交易、会过期 |
| 势力装 | 由势力主题强化的高品质装备，稀有但仍低于遗物层级 |
| 遗物 | 极稀有高价值资产，多来自命名精英、深层密报与事件 Boss |
| 江湖日报 | 每日第一次登录时生成的个性化反馈界面，负责承接世界变化、市场变化和个人成长摘要 |

## 3. 全局统一规则

### 3.1 日周期口径

- 早上 5 分钟：看日报、收离线、补空槽、查挂单。
- 午休 3 分钟：领阶段产出、改一段队列、看帮派情报。
- 晚上 30-90 分钟：匪巢、远征、副本、护镖、劫镖、擂台、帮战择一或组合。
- 睡前 5 分钟：整理战利品、设过夜队列、补保险或护命符。

### 3.2 区域收益梯度

| 区域 | 理论毛收益 | 熟练玩家净收益 | 不熟练玩家净收益 | 说明 |
| --- | --- | --- | --- | --- |
| 高安 | 1.0 | 1.0 | 1.0 | 保底成长、制造、稳定交易 |
| 低安 | 1.25 | 1.15 | 1.0 | 轻风险高周转、护镖与情报前置 |
| 危险区 | 1.7 | 1.35 | 0.9 | 税权、资源腹地、高价值材料 |
| 伏龙外围事件态 | 2.1 | 1.55 | 0.8 | 深层远征与世界事件放大器 |

### 3.3 离线结算总公式

`实际产出 = 基础效率 x 技能系数 x 设施系数 x 区域系数 x 行具系数 x 时长 x 完成系数`

- `技能系数 = 1 + 0.02 x 熟练档位 + 0.03 x 领悟档位`
- `设施系数` 首版建议值 `1.05 - 1.20`
- `区域系数` 首版建议值：高安 `1.0`、低安 `1.1`、危险区 `1.25`
- `行具系数` 首版建议值 `0.95 - 1.15`
- `完成系数` 正常为 `1.0`，异常中断按完成比例折算

### 3.4 死亡与损毁总规则

- 货物与战利品优先结算掉落，装备以损毁和耐久损失为主。
- 低安优先体现“有限惩罚”，危险区优先体现“高价值消耗出口”。
- 保险只赔银两，不返还原物；护命符只保指定槽位，不免耐久。
- 新手保护覆盖前三章和账号前 72 小时，不允许在低安教育阶段把玩家打穿。

### 3.5 数值表达原则

- 全文数值均视为“首版建议值”，不是上线终值。
- 强平衡项必须同时给出固定值和调参区间。
- 所有装备、技能、掉落、活动奖励都必须说明“收益来源”和“消耗出口”。

## 6. 统一命名与文案规范

### 6.1 命名规范

- 技能名采用“二至四字武学名 + 功能尾缀”的形式，如 `断潮式`、`回风斩`、`藏息诀`。
- 心法名采用“典籍/真诀/录/篇”的形式，如 `青蒲调息诀`、`黑砂炼骨篇`。
- 道具名优先使用江湖口吻，不直接出现现代系统词，如用 `护命符` 而不用 `防爆券`。
- 节点名优先由地貌 + 功能 + 气质构成，如 `落雁旧渡`、`黑风外寨`、`朔砂矿市`。

### 6.2 文案规范

- 按钮文案控制在 2-6 字：`前往下一处`、`接下此单`、`挂号护送`、`收下密报`。
- 系统提示保持武侠语气，但不得牺牲可读性：先讲结果，再讲缘由，再给操作提示。
- 稀有掉落、命名精英、世界事件、日报增刊必须有“江湖味”，但配置表字段保持纯技术可读。

### 6.3 交叉引用规则

- 模块文档中第一次提到共享概念时，沿用本总纲字段名。
- 若子文档对总纲数值有补充，视为“细化”而非“推翻”，不得改动全局收益梯度和安全区逻辑。
- 若某条文案同时影响多个模块，应以首次定义的文档为准，其它模块仅做引用。

## 7. 验收口径

- 总纲可以独立说明产品是什么、首发边界是什么、所有模块在哪里。
- 子文档可以独立交给对应负责人拆表、拆状态机、拆界面。
- 技能、道具、掉落、活动、订单、Boss、章节、节点、税率、保险等专有名词在所有文档中保持统一。
- 任何新增细节都不能破坏既有的风险收益梯度和不卖数值的商业边界。

## 8. 后续维护方式

- 当模块有大改动时，先更新模块文档，再回写总纲的目录说明、共享词典或统一口径。
- 当新增内容超出首发边界，如第四幕、沉灯海隙、完整赛季战争，建议单独开 `V2` 模块文档，不直接塞回本轮首发子文档。
- 本总纲建议作为后续所有系统评审的引用入口。
