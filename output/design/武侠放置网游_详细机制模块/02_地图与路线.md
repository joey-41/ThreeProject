# 02 地图与路线

> 主索引：[武侠放置网游_详细游戏机制文档.md](../武侠放置网游_详细游戏机制文档.md)

## 1. 模块目标

- 把“节点江湖”从世界观概念落实为 196 节点、三层密度、四类路线带的可执行地图结构。
- 明确八大区域群在首发中的节点职责、子市场落点、PvE/PvP 热区和帮派政治目标。
- 提供节点、路线、热度、痕迹、税率、子市场和节点簇的统一字段，为地图客户端与服务端落点判定提供依据。

## 3. 核心循环

`查看节点 -> 预估多条路线的耗时、税费与风险 -> 选择移动/运输/驻留目标 -> 落点触发市场/战斗/税率/驻防/痕迹判定 -> 形成新的热度和情报 -> 反过来影响下一次路线判断`

地图设计必须同时满足四件事：

- 主线玩家能看懂去哪、为什么去，以及为什么不是只有一条路。
- 跑商玩家能看懂哪里有差价、哪里可能被截、哪里值得改道。
- 帮派玩家能看懂哪条补给线要守、哪条绕行线能切、哪几个节点值得驻防。
- 美术和 UI 成本随节点数量上升，但只对核心节点做重资产表达，功能节点和轻节点轻量承接玩法。

每个危险区必须同时存在六类组织目标：

1. 主税点
2. 前哨点
3. 仓站点
4. 修理点
5. 观测点

## 5. 公式与数值

### 5.1 路线耗时

`实际耗时 = 基础路程 x 路线系数 x 行具系数 x 轻功系数 x 舟术/驿务修正 x 事件修正`

- `行具系数`：轻行具 `0.92`，镖运具 `1.00`，战旅具 `1.08`，机关具 `1.03`
- `轻功系数`：`1 - 0.005 x 轻功等级`
- `舟术修正`：仅对水道生效，建议区间 `0.92 - 0.98`
- `驿务修正`：仅对多跳订单经停生效，建议区间 `0.94 - 1.00`
- `事件修正`：世界事件封锁时取 `1.10 - 1.30`

### 5.2 热度刷新

- 热度每 5 分钟刷新一次
- 核心重节点热度上限 120，功能节点 90，轻节点 60
- 近 10 分钟无新事件时每 5 分钟自然衰减 12 点

### 5.3 痕迹识别

`识别成功率 = 基础值 35% + 侦听等级 x 0.8% + 同盟复核加成 12% + 节点情报权重 - 目标隐匿修正`
